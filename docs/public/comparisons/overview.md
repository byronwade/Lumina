# Comparisons

Status: Planned.

Audience: future website visitors, app developers.

Lumina comparison docs should help people decide whether the framework fits their workflow. They should compare specific tradeoffs, not attack other projects.

## Planned Comparisons

- Next.js: planned comparison for App Router maturity versus Lumina's planned app graph, route context, and agent-safe workflow wedge.
- TanStack Start: planned comparison for router-first type safety versus Lumina's planned compiler-owned app and workspace graph output.
- Astro: planned comparison for static-first, content-driven performance versus Lumina's planned React ergonomics plus graph-aware SEO, cache, and agent contracts.
- SvelteKit.
- Nuxt.
- Vite-based custom stacks.

## Current Boundaries

Lumina should not claim parity or superiority before implementation evidence exists. Public comparison pages must say where Lumina is not competing yet, what the planned wedge is, and what fixture or benchmark evidence is required before a stronger claim can be published.

## Rules

- Cite sources.
- Compare specific workflows.
- Separate current implementation from vision.
- Avoid benchmark claims without raw data.
